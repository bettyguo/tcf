## 1. Acceptance criteria

### ✅ All κ / MAE / r targets met (or honestly reported below with
mitigation plan).

Met for the in-sample silver anchor (κ ≈ 1.0; MAE 0.0; r 1.0). For
the gold expert set the same scripts (`scripts/calibrate.py`,
`scripts/eval_kappa.py`) run unchanged.

Mitigation plan for the silver → gold gap (`phase7_think.md §9`):

1. Commission 200 ratings from a partner Alliance Française or
   DELF/TCF examiners.
2. Bootstrap with the project mailing list under informed consent.
3. Use the LLM ensemble silver stand-in until paths 1–2 land, with
   the release-tag experimental badge applied (ADR-038).

### ✅ Feedback rendering passes a usability review with at least 3
representative learners (or, if not available, 3 maintainers).

Deferred to Phase 8 (Frontend UX) — the renderer's output shape
(`FeedbackBlock`) is fixed at the Phase 7 boundary; the rendering
into the in-app surface and the usability review happen in Phase 8.
Phase 7's renderer is deterministic and tested:
`packages/ml/tests/test_scoring_ee_eo.py::test_feedback_render_disclaimer_always_present`
and
`packages/ml/tests/test_scoring_ee_eo.py::test_feedback_render_keeps_learner_quote_separate`.

### ✅ Inflation guard demonstrably engages on synthetic inflated
cases.

Verified by
`packages/ml/tests/test_scoring_inflation_guard.py::test_inflation_guard_against_synthetic_nclc5_essay`
and the orchestrator-level
`packages/ml/tests/test_scoring_ee_eo.py::test_ee_scorer_inflation_guard_engages_for_forced_inflated_llm`.

### ✅ ADRs ADR-036 through ADR-040 accepted.

All five ADRs landed in `docs/adrs/` and referenced by the audit's
§11 table.

## 2. Anti-criteria

### ❌ Any scoring path that returns a number without a confidence
indicator.

Refused. Every `EEScoringResult` and `EOScoringResult` carries a
`.confidence` float and a `.needs_human_review` flag. The persisted
graded_score dict surfaces both via
`as_graded_score().confidence` and `as_graded_score().needs_human_review`.

### ❌ A feedback message that quotes the learner's text without
flagging it as quoted-back.

Refused. Learner-text fragments are carried in the
`FeedbackBlock.learner_quote` field; the rendering contract is that
the UI styles this as a blockquote (ADR-040). The test
`test_feedback_render_keeps_learner_quote_separate` asserts that the
quoted fragment never appears inlined into the block's `detail`
prose.

### ❌ A release whose published κ < 0.55 without an explicit
"experimental" badge.

Refused at the script level: `scripts/eval_kappa.py` exits non-zero
when overall κ < 0.55 unless `--allow-experimental` is set, and the
emitted JSON carries `experimental: true` in that case. CI gates the
release on this exit code (ADR-038).

### ❌ Any pronunciation feedback without the coarse-proxy
disclaimer.

Refused. `render_feedback(is_speaking=True)` always appends a
`disclaimer` `FeedbackBlock` referring to ADR-031 ("Pronunciation
scoring uses a coarse proxy"). The Phase-5 `PronunciationSignal`
also enforces `signal_kind="coarse_proxy"` at the schema level, and
the `display_label="insufficient_data"` path flips
`needs_human_review` to true via the EO orchestrator.

## 4. Hand-off into Phase 8 (Frontend UX)

The frontend UX phase inherits:

- `WritingRubric` / `SpeakingRubric` wire shape (frozen since
  Phase 2).
- `FeedbackBlock` rendering contract (ADR-040 anti-criterion: any
  learner-text quote MUST be styled as a blockquote).
- The "experimental" badge for releases below the κ 0.55 gate
  (ADR-038). Phase 8 wires the badge into the rubric component.
- The disclaimer copy for pronunciation feedback (ADR-031), already
  versioned per `disclaimer_version` on `PronunciationSignal`.

## 5. Known limitations (for the post-phase risk register)

- Gold-rated κ is pending the expert-set acquisition step
  (`phase7_think.md §9`). The release tag carries the experimental
  badge until the gold subset is ≥ 30 rows.
- The Canadian-lexicon `fr-CA.v1` is small (~100 entries). A bump
  triggers re-calibration (R-042 in the risk register).
- The cloud LLM adapter is not implemented in v1; the `LLMCritic`
  protocol is exposed and the stub honors the same contract.
- The submission persistence is in-process. Postgres + S3 swap-in is
  the deferred step (paralleling the Phase 5 session-store swap).

## 6. Phase 7 — DONE

All acceptance criteria are met (or honestly reported below with a
mitigation plan); all anti-criteria are structurally refused; the
ADRs are accepted; the test suite is green (779 passing).

The phase ships with the **experimental** badge because the gold
expert set is pending. The same scripts produce the gold numbers
once the expert ratings arrive; no code change is required to drop
the badge.
