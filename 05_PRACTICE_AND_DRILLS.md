# PHASE 5 — Practice & Drill Engines (CO, CE, EE, EO + Pronunciation)

> Goal: working, exam-aligned drill loops for each of the four modules plus a pronunciation feedback pipeline. Each drill type is tested, accessible, and produces interactions that feed back into the learner model.

---

## 1. THINK (produce `phase5_think.md`)

### 1.1 Drill Taxonomy

Each module has a "core drill" (matches the exam) and several supplementary drills (build sub-skills the exam tests indirectly):

| Module | Core drill | Supplementary drills | Rationale |
|---|---|---|---|
| CO | Single-play MCQ | Dictation, shadowing, gap-fill from audio, accent-discrimination | Decompose listening into sub-skills (phonological, lexical, syntactic, discoursal) |
| CE | Timed-passage MCQ | Skimming-with-deadline, vocab-in-context, register identification, summary generation | Exam tests inference + skimming under time pressure |
| EE | 3-task timed write | Sentence-rewriting, connector drills, error-correction-of-own-text, register adjustment | Errors aggregate at predictable patterns |
| EO | 3-task recorded speak | Picture-description, opinion-with-justification, role-play, repair-after-feedback | Most learners under-practice spontaneous production |
| (Cross-cutting) | — | Pronunciation drills (phoneme + prosody), collocations, false-cognates | High ROI for low marginal time |

### 1.2 Why Single-Play Matters for CO

The TCF Canada CO is played **once**. A common preparation failure is letting learners re-listen during practice, which builds confidence but not exam-aligned skill. The default drill mode is single-play; re-listen is permitted only in *review* mode after answering, never during.

### 1.3 The Tension Between Exam-Realism and Diagnostic Value

A pure mock-exam loop maximizes realism but minimizes feedback. A pure drill loop maximizes feedback but drifts from exam shape. Resolution:

- 80% of practice time = drills with rich feedback.
- 20% of practice time = exam-shape sessions with feedback withheld until the end (matches what the exam day will feel like).

The system enforces this ratio in the planner; if a user has been doing only drills for 7 days, the next session is force-converted to exam-shape.

### 1.4 Speaking is the Hardest Channel to Build

EO is the highest-friction practice for solo learners. Three challenges:

1. **No examiner present** → we simulate with a TTS-voiced examiner + 1:1 timer.
2. **No reference recording** → we provide a model NCLC-9 audio response post-attempt, generated by a strong TTS or pulled from a curated library of Alliance Française model responses (where licensable).
3. **Pronunciation feedback is hard to do well** → forced alignment + phoneme-level scoring, but we are explicit that this is a coarse proxy.

Be modest about pronunciation scoring. Over-claiming here is a real risk.

### 1.5 Alternatives for the EO Examiner

- (a) **TTS-only "robot examiner."** Cheap, deterministic, but unrealistic intonation.
- (b) **Curated human-recorded prompt library.** Realistic but limited content, expensive to expand.
- (c) **High-quality TTS (XTTS-v2 with examiner-style voice + SSML for prosody) for prompts + LLM-driven follow-up question generation.** ← chosen
  - Reason: best balance of realism, scalability, and prompt diversity. Follow-ups dynamically generated based on what the candidate said (key for Task 1 Q&A and Task 3 defense).

## 2. DESIGN (produce `phase5_design.md`)

### 2.1 The Drill Loop (canonical state machine)

```
START ─▶ FETCH_NEXT ─▶ PRESENT ─▶ AWAIT_RESPONSE ─▶ GRADE ─▶ FEEDBACK ─▶ UPDATE_MODELS ─▶ DECIDE_CONTINUE
                                                                                                ├─▶ NEXT (loop)
                                                                                                └─▶ END_SESSION ─▶ SUMMARY
```

Each transition is logged with a timestamp; sessions can be paused and resumed within 24h.

### 2.2 CO Drill Modes

#### Core: Single-Play MCQ
- Audio plays once, no replay.
- Question + 4 options visible during audio (mimics exam UI).
- Response timer: 35 min / 39 items = ~54 s/item average; the drill enforces this in "exam-pace" mode but is untimed in "learn" mode.

#### Supplementary: Dictation
- 8–12 s sentences from the item bank.
- Learner transcribes; we measure WER between transcription and ground-truth transcript.
- Errors classified into: spelling, missing-words, extra-words, agreement, register.

#### Supplementary: Shadowing
- Learner listens, then re-speaks the same sentence within 0.5s of audio end.
- ASR transcribes the learner; we compute phoneme alignment score + WPM ratio (target: 0.85–1.10 of source WPM).
- 10 minutes/day mandatory in the default plan — strong evidence for prosody acquisition.

#### Supplementary: Accent Discrimination
- Two short clips, same content (fr-FR vs fr-CA, or fr-FR vs fr-AF, etc.).
- Learner identifies accent of each + difficulty rating.
- Builds CO robustness — the real exam includes varied accents (per §1.4 of master prompt).

#### Supplementary: Gap-Fill from Audio
- Transcript with N gaps; audio plays once; learner types missing words.
- Tests phonological recognition + spelling.

### 2.3 CE Drill Modes

#### Core: Timed-Passage MCQ
- Passage + question visible; 60 s soft target per item.
- Eye-tracking would be lovely; we approximate by tracking scroll position + time-to-first-answer.

#### Supplementary: Skim-and-Scan
- Passage shown for 30 s, then hidden; 3 detail questions follow.
- Builds skimming under time pressure.

#### Supplementary: Vocab-in-Context
- Passage with N highlighted lexemes; learner picks definition from 4 options where 3 are plausible-but-context-wrong (false-friend traps).

#### Supplementary: Register Identification
- Same passage in 3 register-variants (familier/standard/soutenu); learner sorts.

#### Supplementary: Summary Generation
- Passage; learner writes 40-word summary; LLM scores fidelity + coverage + brevity.

### 2.4 EE Drill Modes

#### Core: 3-Task Timed Write
- Task 1: 60 words descriptive message (10 min).
- Task 2: 120 words opinion piece (20 min).
- Task 3: 180 words argumentative essay (30 min).
- Strict word counter; 80–110% of target accepted, outside that → penalty per FEI guidance.

#### Supplementary: Sentence-Rewriting
- Learner is given a sentence at NCLC 7 register; asked to rewrite at NCLC 9 (more sophisticated lexis, complex syntax).

#### Supplementary: Connector Drill
- Cloze test on discourse markers: 1 connector per 25 words (per §1.4 of master prompt); learner picks from the appropriate semantic category (addition/contrast/cause/conclusion).

#### Supplementary: Error-Correction-of-Own-Text
- After the auto-scorer flags errors, the learner re-writes the flagged sentences. Critical: this is where most learning happens in EE.

#### Supplementary: Register Adjustment
- Given a familier-register message, rewrite formal. Vice versa.

### 2.5 EO Drill Modes

#### Core: 3-Task Recorded
- **Task 1** (~3 min): examiner Q&A. TTS prompts; learner records; LLM generates plausible follow-ups based on the learner's last utterance.
- **Task 2** (~3.5 min): describe and compare 2 images/scenarios. Learner has 2 min prep.
- **Task 3** (~3.5 min): argue a position then respond to objections (LLM-generated objections after first response).

#### Supplementary: Picture Description
- 30 s prep + 90 s production. Tests lexical access under pressure.

#### Supplementary: Spontaneous Opinion
- Prompt appears; 5 s to start speaking; 60 s response.

#### Supplementary: Role-Play
- "You are at the boulangerie and they forgot your order." 90 s, with a TTS interlocutor responding.

#### Supplementary: Repair-After-Feedback
- After auto-scoring, the system identifies the lowest-scoring sub-criterion. A targeted micro-drill on that sub-criterion (e.g., subjunctive triggers) runs.

### 2.6 Pronunciation Pipeline

For every spoken interaction:

```
Audio (16kHz mono) → Whisper-large-v3-french → Transcript
                  → Montreal Forced Aligner → phoneme-level timestamps
                  → Reference phoneme sequence (from canonical pronunciation dict)
                  → Phoneme-Error-Rate (PER) + per-phoneme accuracy
                  → Prosody analysis (pitch contour via librosa, stress placement)
                  → SpeakingRubric.pronunciation_prosody score
```

Caveats encoded in the UX: "Pronunciation feedback is a coarse signal. A human examiner judges intelligibility, accent acceptability, and prosody holistically. Use this signal for trends, not for final readiness."

### 2.7 Accessibility Per Drill Type

- **CO drills**: full transcripts available *after* answering; not during (would defeat the drill). Skip-the-drill alternative for Deaf/HoH users that routes them only to CE/EE/EO.
- **CE drills**: dyslexia font; adjustable spacing; high-contrast mode.
- **EE drills**: speech-to-text input as alternative for motor-impaired users.
- **EO drills**: text input as alternative for those who cannot/do not wish to record (with a clear caveat that this does not measure real EO).

### 2.8 ADRs

- ADR-028: 80/20 split between drills and exam-shape practice, enforced by the planner.
- ADR-029: Single-play default for CO; re-listen only in post-answer review.
- ADR-030: Mandatory 10 min/day shadowing in default plans.
- ADR-031: Pronunciation feedback labeled as coarse-proxy, not as evaluative.

## 4. AUDIT (produce `phase5_audit.md`)

- **Correctness:** for each drill type, write a scripted "perfect agent" that always answers correctly; the agent's final session report must show 100% accuracy, expected FSRS state updates, correct posterior shift.
- **Pacing:** the timer in exam-pace mode matches the official module duration to within ±1 s over a 35-min stretch.
- **Pronunciation pipeline end-to-end:** on a 50-utterance held-out test set with hand-labeled phoneme transcriptions, MFA + our pipeline produces a PER within ±0.05 of expert annotations.
- **ASR quality:** WER on Common Voice fr test set ≤ 9% (matches published whisper-large-v3-french benchmarks).
- **Accessibility:** axe-core scan returns zero violations on every drill UI; keyboard-only walkthrough succeeds for every drill.
- **Drill diversity audit:** in a 100-session synthetic run, the planner selects from ≥ 4 drill types per module (no monoculture).

## 5. EVALUATE (produce `phase5_evaluate.md`)

Acceptance criteria:

- ✅ All five drill types per module functioning.
- ✅ Pronunciation pipeline metrics meet target.
- ✅ Accessibility audits clean.
- ✅ Each drill produces a properly-typed Interaction that the Phase 4 estimator consumes.
- ✅ ADRs ADR-028 through ADR-031 accepted.

Anti-criteria:

- ❌ Any drill that allows replay of CO audio during the question.
- ❌ Any pronunciation score presented without the "coarse proxy" disclaimer.
- ❌ Any drill UI where keyboard navigation breaks.
- ❌ Any session that completes without writing the Interaction rows.

Hand-off: a video walkthrough (or detailed README) of each drill type; a synthetic-cohort report showing skill improvement after 100 drill sessions per archetype.
