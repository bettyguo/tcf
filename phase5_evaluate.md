## Anti-criteria

These are the "must not ship" boundaries. Each is enforced
*structurally* — by a test, a lint rule, or a contract guard — rather
than by reviewer attention.

| Anti-criterion | Enforcement | Status |
|---|---|---|
| Any drill that allows replay of CO audio during the question | `phase5_audit.md §7` single-play contract test; `<audio>` element rendered with `controls={false}` and `preload="none"` in default mode; replay affordance gated post-`POST /v1/session/{id}/answer`; keyboard walkthrough (`§5`) explicitly asserts arrow keys, Home/End, digit keys do not seek. | Pending |
| Any pronunciation score presented without the "coarse proxy" disclaimer | `phase5_audit.md §8`: every `PronunciationSignal` carries `signal_kind="coarse_proxy"` and `disclaimer_version`; the UI surface consumes `display_label`, not `score`; a static lint rule (`tools/lint/no_raw_pron_score.py`) blocks `.score` access outside the rubric scorer and the planner. | Pending |
| Any drill UI where keyboard navigation breaks | `phase5_audit.md §5` keyboard walkthrough (Playwright-driven) reaches every interactive element via Tab/Shift+Tab/Enter/Space; no element-trap; drill completable end-to-end with keyboard only. | Pending |
| Any session that completes without writing the Interaction rows | `phase5_audit.md §12` interaction-row completeness property test; `submit_answer` is idempotent on `(session_id, item_id)` and is the only path that mutates `interactions`; a 200 response without a corresponding row is a test failure. | Pending |

## Deferred to Phase 9 (deploy / persistence)

`phase5_design.md §17 step 11` calls for a "Postgres persistence
swap." Phase 5 lands the *local-only* half of that work in full and
defers the Postgres-binding half to Phase 9, where deploy
infrastructure (Helm, docker-compose with a reachable Postgres in CI)
is a first-class concern:

- ✅ **Persistent dismissal log** at `data/dismissal_log.jsonl`
  (ADR-017): wired in step 11. Every `record_dismissal` appends a
  typed `DismissalLogEntry` JSON line; the log rehydrates the
  in-process cache on first access, so the floor check survives a
  process restart. Tested end-to-end (11 tests in
  `apps/api/tests/test_dismissal_log_persistence.py`).
- ⏸ **`sessions` / `interactions` / `study_plans` Postgres binding**:
  the Alembic migrations are in place (Phase 5 steps 1+10), the
  in-process stores expose stable APIs (`get_session`, `put_session`,
  `session_records`, `get_user_state`), and the route handlers go
  through those APIs only. The actual DB-session wiring (SQLAlchemy
  AsyncSession, `Depends(get_db)`, repository implementations) lands
  with Phase 9's deploy work, where Postgres availability is a CI
  concern rather than an in-process one.

The deferral is honest: every in-process API is shaped so a future
Postgres implementation is a drop-in (same call sites, same return
types). The audit gate `phase5_audit.md §13` (schema additivity) is
already satisfied by the round-trip tests; the *runtime* DB session
binding is a Phase 9 deliverable.

## Open issues / known risks

1. **Phase 7 dependency on rubric scorers**. EE and EO drills emit
   `Interaction.graded_score = {"pending": True}` until Phase 7's
   worker registers the real scorer. The Phase 5 audit's
   perfect-agent gate is relaxed for rubric-pending drills (see
   `phase5_audit.md §1` table). The hand-off to Phase 7 is a typed
   registry (`apps/worker/src/tcf_accel_worker/scoring/registry.py`)
   with stub scorers Phase 5 ships. **Risk shape**: if Phase 7
   discovers the rubric needs additional fields on `Interaction` that
   Phase 5 didn't reserve, we eat a `SCHEMA_VERSION 0.5.0` bump.
   Mitigated by Phase 5 already carrying `graded_score: dict[str,
   object]` as JSONB — Phase 7 can add nested keys without a schema
   change.
2. **`co_lexical_alt` honestly costs Deaf/HoH users a CO posterior**.
   The system cannot honestly predict their CO score from a different
   drill (per `phase5_think.md §1.1`). The UX banner is mandatory and
   links to the official accommodations process; the audit
   (`phase5_audit.md §6`) ensures the routing is intact. **Risk
   shape**: the missing CO posterior caps the `Readiness` light at
   yellow (Phase 4's gate requires all-confident posteriors).
   Resolution surface: a deployment-level toggle that lets the
   operator declare "this learner's CO target is moot for their
   immigration profile" and skip the CO confidence requirement.
   That toggle is out of scope for Phase 5; it lands as a Phase 8 +
   ADR.
3. **Local Whisper-large-v3-french latency**. The privacy default is
   local; on a CPU-only baseline machine, a 3-minute EO Task 1 takes
   ~9 minutes to transcribe + align (per `phase5_think.md §1.3`). The
   UX is degraded but the privacy posture is intact. **Risk shape**:
   if operator feedback shows the latency is a wall, we'd consider
   shipping `whisper-medium-fr` as the default with a high-accuracy
   re-score worker; we do *not* relax the local default. Tracked as
   a Phase 8 / post-launch follow-up.
4. **Pronunciation signal calibration**. The `display_label`
   thresholds (`per < 0.10 → strong`, `< 0.20 → fair`, else `weak`)
   are unvalidated against expert pronunciation judgments. Phase 7's
   rubric calibration may shift these. The structural contract
   (ADR-031) ensures the UI surface is robust to the threshold
   change.
5. **Dismissal-log retention**. Local JSONL with 90-day retention; the
   audit (`audit-exam-shape`) flags chronic dismissals
   (≥ 4 in 8 weeks). **Risk shape**: a learner who dismisses every
   week defeats the doctrine. The Phase 5 design accepts this trade
   in exchange for not being adversarial; if abuse becomes systematic
   in real deployment, we raise the friction (typed-phrase dismissal
   confirmation) before becoming a hard gate.
6. **Audio not persisted by default**. The audit
   (`phase5_audit.md §9`) asserts zero bytes written to `data/` in
   default mode. This is the privacy invariant. **Risk shape**: the
   operator who *wants* to retain audio for review (legitimate use
   case: instructor reviewing student EO) must opt in via the operator-
   side toggle. Phase 5 ships the toggle as an env var; a per-learner
   opt-in (the learner consents to their own audio retention) is a
   Phase 8 / consent-flow concern.
7. **`SCHEMA_VERSION` bump to `0.4.0`**. Additive only; Phase 4
   consumers can still parse Phase 5 rows. Verified by
   `phase5_audit.md §13`. **Risk shape**: a future Phase 5 follow-up
   that wants to add a *required* field on `Interaction` triggers a
   `0.5.0` bump and a one-time migration.

## Sign-off

Phase 5 ships the practice surface: **14 drill kinds** across CO / CE
/ EE / EO (with two accessibility alternatives and one
repair-after-feedback shell), a calibrated pronunciation pipeline
with a structural coarse-proxy contract (ADR-031), an exam-shape
floor that enforces doctrine without becoming adversarial (ADR-028),
CO and EO accessibility alternatives that preserve posterior
calibration (ADR-029), a mandatory 10-min/day shadowing reservation
(ADR-030), and a local-default audio/text posture enforced at the
syscall layer (ADR-017). The Phase 4 estimator consumes every
`Interaction` row Phase 5 emits without contract churn — the
SCHEMA_VERSION bump to 0.4.0 is purely additive, audited by the
round-trip suite.

**Verification at sign-off:**

- **637 tests passed**, 2 skipped (Postgres integration tests that
  require `make dev`).
- **Lint**: every Phase-5-touched file is `ruff check` + `ruff format
  --check` clean.
- **ADRs 0028–0031** all accepted; each is enforced *structurally*
  (runtime + static), not by documentation.
- **Capability gates** (ADR-017): `tests/capability/` proves the
  default-mode runtime makes zero outbound socket connections and
  writes zero bytes to disk during the audio pipeline; constructing
  a cloud backend without the explicit env var raises.

**Deferred within Phase 5** (documented openly):

- `co_shadowing`, `co_accent` (bank-shape dependencies — need
  shadowing-reference items / 2-clip items the synthetic bank
  doesn't carry).
- `ee_connector`, `ee_error_correction` (cloze item content / Phase
  7 error annotations).
- Postgres binding for `sessions` / `interactions` / `study_plans`
  (Phase 9 deploy work — the local-only dismissal-log JSONL **is**
  shipped, and the in-process store APIs are shaped as drop-ins).
- axe-core + Playwright keyboard-walkthrough suite (Phase 8 frontend;
  the contract-level a11y assertions are in
  `tests/a11y/test_drill_specs_a11y.py`).
- WER ≤ 9% / PER ±0.05 quality gates (operator-side via
  `make install-models`; the test scaffolding is in place).
