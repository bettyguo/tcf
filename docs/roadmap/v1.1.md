# v1.1 roadmap

> The "what's missing in v1.0" list, pre-empted. Each item is
> sequenced for v1.1 (or later) and *deliberately not in v1.0*.
> Phase 9 §2.8 of `09_QUALITY_AUDIT_AND_LAUNCH.md` motivates this
> doc.

The point of this list: a reviewer asking "what's missing?" can
read it here and know we have a plan, not that we forgot.

---

## v1.1 — within ~6 months of v1.0

### 1. Native mobile (PWA today; native via Tauri + Capacitor next)

**Why deferred:** the web PWA installs from `/today` and covers the
v1 mobile case. We don't have native testers to ship native apps
honestly at v1.0.

**Design sketch:** Tauri for desktop (macOS/Linux/Windows);
Capacitor or React Native for iOS/Android. Reuses the existing
Next.js components and the `/v1/*` API.

**Scope guard:** if native ships, the privacy posture (local_only
default, no analytics) is preserved.

### 2. Tutor / multi-tenant deployment for Alliance Française / language schools

**Why deferred:** we haven't designed the auth sharding. v1.0 is
single-tenant-per-learner.

**Design sketch:** a `tenant` column on every user-scoped table; a
JWT claim that carries the tenant id; row-level security at the
Postgres level; a `tutor` role that can read aggregated cohort
stats but not individual learner trajectories.

**Scope guard:** the data-isolation surface must be fuzzed twice as
aggressively as the v1.0 user-isolation surface.

### 3. TEF Canada parallel track

**Why deferred:** ~80% of the bank applies; ~20% needs re-leveling
+ a different rubric for the production sections.

**Design sketch:** an `exam_format` field on the user; the planner
+ mocks branch on the field. The pedagogy audit cohort matrix
doubles.

**Scope guard:** v1.1 adds TEF Canada; DELF/DALF wait for v1.2.

### 4. Improved κ (target ≥ 0.75 with a larger expert set)

**Why deferred:** the acquisition challenge is the 200-row gold set
per R-010. We need a partner (Alliance Française, DELF examiners)
to obtain it.

**Design sketch:** ADR-038's "experimental" badge drops once we
publish κ_gold ≥ 0.75 on EE and EO.

**Scope guard:** we don't claim the higher κ until the dataset is
public + auditable.

### 5. Better Canadian-context coverage

**Why deferred:** the `fr-CA.v1` lexicon is ~100 entries (R-042).
We need a Québec educator partner to expand.

**Design sketch:** `fr-CA.v2` lexicon + Quebec-specific cultural
items + a regional-register classifier feature.

**Scope guard:** any lexicon bump is an ADR-grade change (R-042
mitigation).

### 6. A1 / pre-B1 onboarding (12-week pre-foundation arc)

**Why deferred:** v1.0 assumes B1 baseline. Below-B1 learners
currently get a "consider a foundation course first" message.

**Design sketch:** a 12-week pre-foundation arc that lands the
learner at solid B1 in time to start the standard 12-week TCF
arc. Total program becomes 24 weeks.

**Scope guard:** the pre-foundation arc is opt-in and clearly
labelled "before TCF prep, not TCF prep."

## v1.2 — within ~12 months of v1.0

### 7. DELF / DALF support

A larger lift — different rubric, no direct NCLC mapping.
Multi-month design phase.

### 8. Online-cohort feature (opt-in study groups, pseudonymous)

Requires real-time infrastructure we have not built. Pseudonymous
by design (no real names, no avatars from social platforms).

### 9. External security audit budgeted

ADR-047 commits v1.1 to acquiring an external audit if budget
permits. v1.0 ships with the two-maintainer fallback.

---

## Permanently out of scope

Things we will not do, ever, in this project:

- **Streaks, leaderboards, gamification surfaces.** ADR-0042 binds
  this. Coercive UI patterns produce study under duress.
- **Pass guarantees.** ADR-046 binds this implicitly via the launch
  criteria; we will not market what we cannot deliver.
- **Real-name social features.** Pseudonymous study groups are the
  ceiling.
- **Selling learner data to third parties.** The license doesn't
  permit it; the project's posture won't permit it.

## How the roadmap is maintained

Every PR that proposes a new v1.x feature must update this file
under the appropriate section. Items get *removed* from the
roadmap only when they ship or when an ADR formally retires them.

The maintainer review for the v1.1 announcement uses this file as
the press-release source.
